## Workflow

Six commands organized by product lifecycle stage. Full details in `.claude/ARCHITECTURE.md`.

**Vision** (run once, occasionally refreshed):
- `/sb-init` — heavy four-lens intake (business, user, product, technical). Produces STRATEGY.md, ARCHITECTURE.md, ROADMAP.md. Add `--quick` for a 6-question fast path producing only STRATEGY.md.

**Per feature** (the loop):
- `/sb-plan <feature>` — five-stage pre-execution pipeline with lane inference (Tiny / Small / Normal / High-risk). Add `--tiny`, `--quick`, or `--high-risk` to override the inferred lane.
- `/sb-work` — execute the approved plan. Produces mandatory divergence summary at end.
- `/sb-review` — multi-perspective code review (now five passes: divergence check + four standard).

**After ship:**
- `/sb-eval <feature>` — measure against named metrics. Default window 30 days; `--window 7d` for shipping-quality check.

**Context hygiene** (anytime):
- `/sb-sanity` — fresh-agent drift check across all artifacts. Add `--with-handoff` for orientation snapshot too.

## Decisions

When agents face judgment calls they can't resolve, they escalate to you. Decisions get logged to `docs/decisions/` and future agents read them as precedent. Just `ls docs/decisions/` to browse.

## Where things live

- `docs/STRATEGY.md` — the bet, the user, metrics, red lines, MVP scope (created by `/sb-init`)
- `docs/ARCHITECTURE.md` — components, data flow, technical risks (deep mode only)
- `docs/ROADMAP.md` — feature horizons with parallelizability annotations (deep mode only)
- `docs/plans/` — one plan per feature
- `docs/reviews/` — code review findings
- `docs/evals/` — post-ship evaluations
- `docs/decisions/` — resolved escalations (precedent for future agents)
- `docs/audits/` — auditor findings from various command stages
- `docs/HANDOFF.md` — project state snapshot (regenerated by `/sb-sanity --with-handoff`)
